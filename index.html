Introduction

Deploy to cPanel with FTP is an automated workflow that allows you to deploy your code to a cPanel hosting server using FTP. This workflow is triggered on each push to the master branch.

Usage

To use this workflow, add the following YAML code to your GitHub Actions workflow file (e.g., .github/workflows/deploy.yml): 


name: Deploy to cPanel with FTP

on:
  push:
    branches:
      - master

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout repository
        uses: actions/checkout@v2

      - name: FTP Deploy
        uses: SamKirkland/FTP-Deploy-Action@v4.3.4
        with:
          server: ${{ secrets.FTP_SERVER }}
          username: ${{ secrets.FTP_USERNAME }}
          password: ${{ secrets.FTP_PASSWORD }}
          server-dir: "/htdocs/"

Configuration

Before using the workflow, make sure to set up the following secrets in your repository settings:

  • FTP_SERVER: The address of your cPanel hosting server.
  • FTP_USERNAME: Your FTP username.
  • FTP_PASSWORD: Your FTP password.

Once you have configured the secrets, the workflow will automatically deploy your code to the specified cPanel directory whenever you push to the master branch.
